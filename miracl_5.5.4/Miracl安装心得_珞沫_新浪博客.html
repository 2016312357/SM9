Miracl安装心得

(2014-04-03 14:59:28)
转载▼

Linux下采用eclipse进行程序编译和运行：

主要介绍Miracl，Crypto++（needed by Miracl）以及OpenSSL的安装和测试中遇到的相关问题。

Miracl：

关于miracl下Linux下的安装步骤（安装包解压之后，有linux.txt，安装步骤如下三步）：

1. Unzip the MIRACL.ZIP file using the utility unzip, into an empty directory

 

unzip -j -aa -L miracl.zip

 

The -j ignores the directory structure inside MIRACL.ZIP. The -aa converts all

text files to Unix format, and -L ensures that all filenames are lower-case.

（-j 命令将忽略文件夹之间的目录结构，直接全部放在一个文件夹里面）

 

2. Perform a tailored build of the MIRACL library by opening a terminal

window, and typing

 

bash linux

（bash linux会出现很多问题。但是经常是缺失很多相关组件，直接采用apt-get进行安装。

还有一个问题就是，如果在第一步中你没有使用-j也可能出现很多问题，你可能需要的相关东西找不到，具体bash linux的含义不太懂，需要的具体东西也不太清楚。所以如果对linux不了解的话，最好还是按教程进行解压缩。 最后也还是会有cannot remove *.exe的问题，但是没事。）

 

3. All the MIRACL applications (except RATCALC) can then be built, as desired. Remember to link all C applications to the miracl.a library. C++ applications must be linked as well to one or more of big.o zzn.o ecn.o crt.o flash.o object files etc.

（miracl是c程序的，在Linux下用c++编写代码的时候，需要先将部分.cpp的文件进行编译生成.o文件。然后才能获取相关命令。比如，我使用了big.cpp中的相关定义，于是我可以将big.cpp和big.h文件加到工程里面，然后直接进行编译。

也可以运用ar命令将big.o等相关的目标文件添加到静态库liamiracl.a中。）

 

接下来是miracl在编译使用中的问题。

首先是libmiracl.a的链接，在eclipse中，添加lib路径和文件的时候。路径选对，然后添加文件，是直接填写miracl，而不要后缀名和lib的。因为在编译中命令是-lmiracl，无需后缀或者lib。

其次，在miracl的时候，向来是需要libdl.a的。同样，你只需要进行添加路径和dl就好。dl库的头文件添加是#include 。

最后还有一个就是在库的包含中，顺序是有关系的。

 

最初我将dl放在最上面，会出现

undefined reference to `dlopen'

 undefined reference to `dlsym'

 undefined reference to `dlsym'类似的错误。可能dl需要ssl和crypto的支持吧，这是乱说的。个人不太懂。

其中ssl和crypto是Openssl安装时，一起生成的。好像顺序也有讲究。开始顺序放置相反时，一直报ssl中的相关错误。

Cryptopp是miracl所需要的，我有特意放在前面。不知道有没有关系。

 

关于miracl一个简单程序的说明。

 

#include

#include "big.h"  

using namespace std;

Miracl precision(500,10); // This makes sure that MIRACL

                         // is initialised before main()  // is called

int main()

{

    Big nf=1;      

    int n;

    cout << "factorial program\n";

    cout << "input number n= \n";

    cin >> n;

    while (n>1)

        nf*=(n--); 

    cout << "n!=\n" << nf << "\n";

    return 0;

}

采用C++编写。最初运行的时候，就因为big.o的问题，一直报错undefined reference to `operator<<(std:stream&, Big const&')。后来就是将big.cpp和big.h添加到工程进行编译，才获得了相关operator的重载定义。

其实在OpenSSL和Crypto++安装中也遇到了很多问题，但是无非就是路径的相关设置，以及头文件的问题。

因为自己完全是个菜鸟，所以很多东西都不太懂，查找了很多东西，但是好像不知道是因为太简单还是怎么的，在网上很多问题找不到解答。所以自己将经验帖出来。希望后面有和我一样完全不懂的人可以快速解决相关问题。

如有问题，欢迎指正，愿我可以学到更多。
